## 1) Project Overview

**Goal:** classify handwritten characters (A–Z) using a from-scratch **Multi-Layer Perceptron (MLP)** implemented with **NumPy**, and provide an interactive **Tkinter GUI** for training and prediction.

**Key parts:**

- `train_mlp.py` trains/evaluates the MLP and saves plots + model weights.
- `gui_app.py` provides an interactive interface to train, load, and predict from drawings/images.
- `src/ann/mlp.py` is the full from-scratch neural network implementation.
- `src/ann/data.py` loads NPZ datasets and performs train/validation/test splits.
- `src/ann/utils.py` contains one-hot encoding and evaluation utilities.

**Deliverables (artifacts produced by this repo):**

- Trained model weights: `models/mlp_az.npz` (CLI) and `models/gui_mlp_model.npz` (GUI training output)
- Training curves: `outputs/mlp_training_curve.png`
- Confusion matrix visualization: `outputs/mlp_confusion_matrix.png`
- Report figures: logic-gate plots and conceptual NN diagrams in `outputs/` (generated by `generate_report_assets.py`)

**Scope / constraints:**

- Intentionally “from scratch” for learning and submission: NumPy-based forward/backprop, no TensorFlow/PyTorch.
- Focused on a straightforward baseline (MLP) and a usable GUI workflow.

**Known limitations (and why they matter):**

- An MLP on raw 28×28 pixels is typically outperformed by CNNs on image tasks; many errors correspond to visually similar letters (see the confusion matrix).
- The dataset is imbalanced (some classes have ~4.5× more samples than others), which can bias predictions toward frequent classes.
- No augmentation (rotations, shifts, thickness changes) is applied during training in this baseline.

## 3) Dataset and Preprocessing

### Dataset format (.npz)

The primary dataset is:

- `character_fonts (with handwritten data).npz`

Expected NPZ keys:

- `images`: shape `(N, H, W)` (grayscale, uint8)
- `labels`: shape `(N,)` (integer class labels)
- optional `class_names`: shape `(K,)` array of strings (class label names)

Observed dataset statistics (from the copy included in this workspace):

- NPZ keys present: `images`, `labels` (no `class_names` key)
- `images`: `(762213, 28, 28)`, dtype `uint8`
- `labels`: `(762213,)`, dtype `uint8`
- classes: 26 (treated as A–Z after remapping to contiguous `0..25`)
- class imbalance (counts per class): min 16,109; max 72,816; mean 29,315.88

### Input representation

The model expects:

- **28×28 grayscale** (by default)
- normalized floats in `[0, 1]`
- flattened to a 1D vector of length `28*28 = 784`

### Preprocessing in GUI

The GUI uses a robust `_prepare_image_for_model` step:

- converts to grayscale
- estimates whether the foreground is dark or light
- crops around “ink pixels” using a mask
- centers the cropped character in a padded square
- autocontrast
- resizes to 28×28

This improves prediction quality by aligning the input distribution with the training data.

## 4) Neural Network Design and Math

### Architecture

This is a standard dense MLP:

- Input: 784 features
- Hidden layers: configurable (default 256, 128)
- Output: `K` classes (typically 26)

Shipped model configuration (`models/mlp_az.npz`):

- architecture: **784 → 256 → 128 → 26**
- hidden activation: **ReLU**
- parameters: **237,210** total (weights + biases)
- saved training hyperparameters (stored in the model file): learning rate ≈ **0.01**, epochs **20**, batch size **256**

Each layer computes:

- Linear transform: $z = aW + b$
- Hidden activation: $a = \text{ReLU}(z)$ or $\sigma(z)$

Output layer uses **softmax**:

$$\text{softmax}(z)_i = \frac{e^{z_i}}{\sum_j e^{z_j}}$$

### Loss function

Cross-entropy for one-hot labels:

$$L = -\frac{1}{m}\sum_{n=1}^m \sum_{k=1}^K y_{n,k} \log(\hat{y}_{n,k})$$

### Backpropagation core idea

- compute output error $\delta^{(L)} = \hat{y} - y$
- propagate backward through weights and activation derivatives
- update parameters with gradient descent:

$$W \leftarrow W - \eta \nabla_W L,\quad b \leftarrow b - \eta \nabla_b L$$

This repo implements backprop “by hand” with NumPy, without TensorFlow/PyTorch.

## 6) Code Walkthrough — Every Function

This section documents each file and every function/method defined in it.

### 6.1 `src/ann/utils.py`

#### Constants

- `VOWEL_LABELS`: a set of numeric label IDs treated as vowels `{A,E,I,O,U}` assuming 0-based A–Z indexing.

#### Functions

- `one_hot_encode(labels: np.ndarray, num_classes: int) -> np.ndarray`
  - Creates a `(N, num_classes)` float32 matrix where each row has a single 1.0 at the label index.
  - Used by the MLP loss and training loop.

- `labels_to_vowel_consonant(labels: np.ndarray) -> np.ndarray`
  - Converts A–Z labels into a binary vowel/consonant label (1 for vowel, 0 for consonant).
  - Not used by the main training script, but useful for experiments.

- `accuracy_score(y_true: np.ndarray, y_pred: np.ndarray) -> float`
  - Returns the mean accuracy.

- `classification_report(y_true: np.ndarray, y_pred: np.ndarray, num_classes: int) -> str`
  - Computes per-class precision/recall/F1 and an overall accuracy line.
  - Implemented without external ML libraries.

- `confusion_matrix(y_true: np.ndarray, y_pred: np.ndarray, num_classes: int) -> np.ndarray`
  - Builds a `num_classes × num_classes` matrix where `(i,j)` counts true=i predicted=j.

### 6.2 `src/ann/data.py`

#### Private helper

- `_resize_images(images: np.ndarray, target_size: int) -> np.ndarray`
  - Resizes a batch of images to `target_size × target_size` using PIL bilinear resampling.
  - Only runs if images are not already the right shape.

#### Public functions

- `load_from_npz(npz_path, target_size=28, flatten=True, normalize=True) -> (x, labels)`
  - Loads `images` and `labels` from an NPZ.
  - Resizes images to `target_size`.
  - Converts to float32 and optionally normalizes to `[0,1]`.
  - Optionally flattens images into vectors.

- `get_npz_class_names(npz_path) -> list[str] | None`
  - If the NPZ contains `class_names`, returns them as Python strings.
  - Otherwise returns `None`.

- `make_labels_contiguous(labels, class_names=None) -> (remapped_labels, names)`
  - Ensures labels become contiguous integers `0..K-1` (important for one-hot encoding and output layer size).
  - If `class_names` are available and match the number of unique labels, they are used.
  - If labels look like `0..25`, it auto-names classes `A..Z`.
  - Otherwise it names classes by their unique numeric labels.

- `load_from_image_folders(root_dir, target_size=28, flatten=True, normalize=True, max_per_class=None) -> (x, y)`
  - Loads images from per-class directories (A/, B/, …).
  - Handles the case of nested folders by scanning for uppercase single-letter folders.
  - Returns normalized and optionally flattened arrays.

- `train_test_split(x, y, test_size=0.2, seed=42, stratify=True) -> (x_train, x_test, y_train, y_test)`
  - Splits arrays into train/test.
  - If `stratify=True`, preserves per-class proportions by splitting each class separately.

### 6.3 `src/ann/mlp.py`

This file contains the full neural network implementation.

#### Data structures

- `MLPHistory`
  - `losses`, `accuracies`: training metrics per epoch
  - `val_losses`, `val_accuracies`: validation metrics per epoch (if validation is enabled)
  - `best_epoch`: epoch index (1-based) with the best validation loss

#### Class: `MLPClassifier`

##### `__init__(layer_sizes, learning_rate=0.01, epochs=20, batch_size=128, hidden_activation='relu', seed=42, class_names=None)`

- Validates that `layer_sizes` includes at least input and output.
- Initializes weights and biases for each layer.
- Uses He-like scaling for ReLU (`sqrt(2/fan_in)`) and Xavier-like scaling for sigmoid (`sqrt(1/fan_in)`).

##### `_hidden_activation(x) -> np.ndarray`

- Applies the chosen hidden activation:
  - sigmoid: $\sigma(x)=1/(1+e^{-x})$
  - ReLU: $\max(0,x)$

##### `_hidden_activation_derivative(x) -> np.ndarray`

- Computes derivative of the hidden activation for backprop:
  - sigmoid derivative: $\sigma(x)(1-\sigma(x))$
  - ReLU derivative: 1 where $x>0$, else 0

##### `_softmax(x) -> np.ndarray` (static)

- Numerically stable softmax by subtracting `max` per row before exponentiation.

##### `_cross_entropy(y_true_one_hot, y_pred_probs) -> float` (static)

- Computes mean cross-entropy loss.
- Uses clipping to avoid `log(0)`.

##### `forward(x) -> (activations, pre_activations)`

- Executes a full forward pass.
- Returns:
  - `activations`: list of layer outputs including input and final probabilities
  - `pre_activations`: list of raw linear outputs `z` per layer

##### `_backward(activations, pre_activations, y_true_one_hot) -> (grad_w, grad_b)`

- Computes gradients for all layers using backprop.
- Output layer gradient uses the softmax + cross-entropy simplification: `delta = y_hat - y`.

##### `fit(x, y, x_val=None, y_val=None, early_stopping_patience=None, verbose=True, epoch_callback=None) -> MLPHistory`

- Trains the model using minibatch gradient descent.
- Shuffles data each epoch.
- Computes epoch loss/accuracy efficiently by accumulating minibatch contributions.
- Optional validation:
  - evaluates on `(x_val, y_val)` each epoch
  - tracks best validation loss
  - supports early stopping based on `early_stopping_patience`
- Optional callback:
  - `epoch_callback(epoch, total, metrics)` enables the GUI to update live plots safely.

##### `predict_proba(x) -> np.ndarray`

- Returns class probabilities for each sample.

##### `predict(x) -> np.ndarray`

- Returns argmax class indices.

##### `evaluate(x, y) -> dict[str, float]`

- Computes loss and accuracy on a dataset.

##### `save(path) -> None`

- Saves model configuration and learned parameters into a `.npz` file.
- Stores weights as `w_0, w_1, ...` and biases as `b_0, b_1, ...`.
- Includes `class_names` when available.

##### `load(path) -> MLPClassifier` (class method)

- Loads a `.npz` model file created by `save()`.
- Reconstructs the `MLPClassifier` and replaces weights/biases.

### 6.4 `train_mlp.py`

This is the command-line training entry point.

#### Functions

- `parse_args() -> argparse.Namespace`
  - Defines CLI arguments such as epochs, hidden sizes, batch size, learning rate, activation, and output paths.

### 6.5 `gui_app.py`

This provides the full interactive application.

#### Class: `ANNGui(tk.Tk)`

##### `__init__(self)`

- Creates the main window and stores UI state in Tkinter variables.
- Initializes drawing state and an in-memory PIL image for the canvas.
- Creates a worker thread slot and an event queue to safely communicate training progress.
- Builds the UI, sets the close handler, and starts polling the queue using `after()`.

##### `_build_style(self)`

- Configures ttk theme and widget styles.
- Sets background and button colors.

##### `_build_ui(self)`

- Builds the window layout using a horizontal PanedWindow.
- Left side: dataset selection, hyperparameters, training buttons, progress bar, status log.
- Right side: notebook tabs for drawing, training metrics, and probability view.

##### `_build_draw_tab(self, parent)`

- Creates the drawing canvas and binds mouse events.
- Adds brush-size control, clear button, and “Predict Drawing” button.
- Adds prediction details:
  - predicted label + confidence
  - processed 28×28 preview
  - a tree view listing top predictions

##### `_build_metrics_tab(self, parent)`

- Builds a Matplotlib figure with two subplots for loss and accuracy.
- Embeds it in Tkinter via `FigureCanvasTkAgg`.

##### `_build_probability_tab(self, parent)`

- Builds a Matplotlib figure for probability bars.

##### `_browse_dataset(self)`

- Opens file picker for `.npz` datasets and sets the dataset path.

##### `_load_model(self)`

- Opens file picker for `.npz` models.
- Loads the model via `MLPClassifier.load()`.
- Extracts `class_names` and estimates dataset polarity.

##### `_append_status(self, message)`

- Appends text to the status log.
- Keeps the widget from growing without bound by trimming old lines.

##### `_queue_event(self, event, payload)`

- Pushes `(event, payload)` onto the thread-safe queue.
- Used by the worker thread to request UI updates.

##### `_process_ui_events(self)`

- Periodically called on the Tk main thread (`after(80, ...)`).
- Drains up to 80 events per tick.
- Routes each event to `_handle_ui_event`.

##### `_on_close(self)`

- Cancels the queue polling callback if present.
- Closes Matplotlib figures.
- Destroys the window.

##### `_handle_ui_event(self, event, payload)`

Handles UI events emitted by the worker thread:

- `status`: append status message
- `dataset_polarity`: store whether dataset background is dark
- `train_start`: reset metrics and configure progress bar
- `epoch`: update progress + curves + status message
- `train_complete`: store model, enable buttons, show final metrics
- `train_error`: show error message in UI and in a messagebox

##### `_parse_hidden_layers(self) -> list[int]`

- Parses the text field `"256,128"` into `[256, 128]`.
- Validates values are positive integers.

##### `_collect_train_config(self) -> dict[str, object]`

- Reads UI fields and validates them.
- Returns a dictionary that can be passed to the worker thread.

##### `_estimate_dataset_dark_background(self) -> bool | None`

- Samples the dataset mean intensity to guess whether background is dark.
- This helps decide whether to invert inputs for prediction.

##### `_train_clicked(self)`

- Validates inputs.
- Disables “Train” button.
- Starts a background thread that runs `_train_worker`.

##### `_train_worker(self, cfg)`

Runs on a background thread:

- Loads and preprocesses dataset
- optionally subsamples
- splits train/val/test
- creates and trains `MLPClassifier`
- uses `epoch_callback` to emit progress events
- evaluates test metrics
- saves the GUI model as `models/gui_mlp_model.npz`
- emits `train_complete` event

##### `_predict_image(self)`

- Opens file picker.
- Loads an image and preprocesses it.
- Runs prediction and updates the UI.

##### `_prepare_image_for_model(self, image) -> Image.Image`

- Converts to grayscale.
- Detects foreground/background polarity.
- Crops to bounding box of foreground pixels.
- Centers and pads to square.
- Autocontrasts and resizes to 28×28.

##### `_run_prediction(self, prepared, source)`

- Computes probabilities on both:
  - original input
  - inverted input
- Picks the better one based on confidence heuristics.
- Updates:
  - predicted label text
  - top-k table
  - probability bar plot
  - 28×28 preview

##### `_update_top_predictions(self, probs)`

- Refreshes the Treeview with the top class probabilities.

##### `_update_preview(self, prepared)`

- Upscales the 28×28 image for display.
- Colorizes for nicer visualization.

##### `_draw_empty_probability_plot(self)`

- Shows an instructional placeholder.

##### `_update_probability_plot(self, probs)`

- Draws a horizontal bar chart of the top N probabilities.

##### `_reset_training_history(self)`

- Clears the stored loss/accuracy lists.

##### `_refresh_training_plot(self)`

- Updates the loss/accuracy plots.
- Shows placeholder text when no training has occurred yet.

##### `_label_for_prediction(self, pred) -> str`

- Resolves class index to:
  - model-provided `class_names`, else
  - dataset `class_names`, else
  - A–Z if output size is 26, else
  - string form of the index

##### Drawing handlers

- `_clear_drawing(self)`
  - Clears the Tk canvas and resets the backing PIL image.

- `_on_draw_start(self, event)`
  - Starts a stroke; draws a dot (oval) on Tk canvas and PIL image.

- `_on_draw_move(self, event)`
  - Draws a line segment from last point to current point.

## 7) Reproducibility Notes and Common Pitfalls

- The NPZ dataset can be very large (often too large for GitHub’s normal file limits). Consider Git LFS or ignoring it in source control.
- GUI training uses a background thread. UI updates are safely delivered by a `Queue` and processed via `after()` on the main thread.
- If predictions look “inverted” (white ink on black background vs black ink on white), the GUI’s prediction logic tries both original and inverted images and chooses the higher-confidence result.

### Verify the baseline numbers in this report

The “Baseline evaluation (shipped model)” section is computed from the files already in the repo (`character_fonts (with handwritten data).npz` + `models/mlp_az.npz`). To re-compute those metrics locally, run Python and execute:

```python
from pathlib import Path

from src.ann.data import get_npz_class_names, load_from_npz, make_labels_contiguous, train_test_split
from src.ann.mlp import MLPClassifier

npz_path = Path('character_fonts (with handwritten data).npz')
model = MLPClassifier.load('models/mlp_az.npz')

x, y = load_from_npz(npz_path, target_size=28, flatten=True, normalize=True)
y, _ = make_labels_contiguous(y, get_npz_class_names(npz_path))
_, x_test, _, y_test = train_test_split(x, y, test_size=0.2, seed=42, stratify=True)

print(model.evaluate(x_test, y_test))
```
